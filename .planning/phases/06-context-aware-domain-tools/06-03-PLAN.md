<objective>
Make `classify_intent` and `extract_params` context-aware by reading `last_turn_context` from GraphState.

Purpose: These two nodes are the core of the follow-up detection mechanism. Without this change, thin follow-ups like "Which of these know Python?" get thin embeddings → low cosine similarity → LLM fallback every time. With this change, they fast-path to the domain tool with the correct intent inherited from the prior turn. This is requirements CTX-02 and CTX-03.

Output: `_is_refinement_followup()` function in classifier, fast path branch in `classify_intent()`, param carry-forward + `_refine_mode` flag in `extract_params()`, full test suite covering detection/rejection cases.
</objective>

```python
    # RBAC gate ...
```

    ```python
    _FOLLOWUP_PATTERNS = re.compile(
        r"\b(?:"
        r"which\s+of\s+th(?:ese|ose)"
        r"|among\s+th(?:em|ese|ose)"
        r"|from\s+th(?:em|ese|ose)"
        r"|of\s+th(?:ose|ese)"
        r"|filter\s+(?:by|them)"
        r"|only\s+th(?:ose|ese)"
        r"|same\s+ones"
        r"|th(?:ose|ese)\s+who"
        r")\b",
        re.IGNORECASE,
    )

    _REFINEMENT_KEYWORDS = re.compile(
        r"\b(?:skill|know|python|java|javascript|typescript|dotnet|\.net|react|angular|"
        r"who|filter|only|active|inactive|billable|assigned|available|unassigned)\b",
        re.IGNORECASE,
    )


    def _is_refinement_followup(question: str, last_turn_context: dict | None) -> bool:
        """Return True if question is a thin follow-up that should inherit prior intent."""
        if not last_turn_context:
            return False
        if not _FOLLOWUP_PATTERNS.search(question):
            return False
        if not _REFINEMENT_KEYWORDS.search(question):
            return False
        return True
    ```

    # Param inheritance:
    - Start params dict with COPY of last_turn_context["params"] (if available)
    - Then extract from current question and OVERLAY (newly extracted wins)
    - This means: if prior turn had skill="Python" and new question adds nothing, params["skill"] = "Python" still
    - If prior turn had skill="Python" and new question says "filter by Java", params["skill"] = "Java"
    
    # Refine mode flag:
    - Set params["_refine_mode"] = True when:
      * last_turn_context is not None
      * last_turn_context["sql"] is truthy (non-empty)
      * state["intent"] == last_turn_context["intent"] (same intent = refinement, not intent switch)
    - Also set params["_prior_sql"] = last_turn_context["sql"] (for domain tool to use as subquery)
    - Also set params["_prior_columns"] = last_turn_context.get("columns", [])
    - If intent differs from prior (intentional topic switch), do NOT set _refine_mode — carry forward params but run fresh intent

    # Test cases:
    - Prior params {"project_name": "Alpha"} + question "Who knows Python?" → params has both project_name="Alpha" and skill="Python"
    - Prior params {"skill": "Python"} + question "Filter by Java" → params["skill"] = "Java" (new wins)
    - Prior context with sql + same intent → _refine_mode=True, _prior_sql set
    - Prior context with sql + DIFFERENT intent → _refine_mode NOT set (not a refinement, it's a switch)
    - No last_turn_context → existing behavior unchanged (params only from current question)
  </behavior>
