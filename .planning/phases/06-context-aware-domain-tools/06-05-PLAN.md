<objective>
Wire `TurnContext` tracking into the chatbot-frontend: types, API client, ChatWidget session management, ChatPanel, and StandaloneChatPage.

Purpose: Without the frontend sending `last_turn_context`, the backend's new follow-up detection (Plan 03) never fires — the state starts as None every request. This plan closes the loop so the full E2E flow works: user asks → response contains turn_context → next question sends it back → backend detects follow-up → subquery refinement runs. Requirements CTX-05, CTX-06, CTX-07.

Output: TypeScript TurnContext type, updated queryApi, ChatWidget with session management, ChatPanel sending session_id + last_turn_context, StandaloneChatPage with lastTurnContext state. All TypeScript strict — no `any`.
