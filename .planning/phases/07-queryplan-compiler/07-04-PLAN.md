  <action>
    Create `backend/app/llm/graph/nodes/semantic_resolver.py`:

    1. `resolve_glossary_hints(db: AsyncSession, connection_id: str, domain: str) -> list[str]`:
       - Query glossary terms from DB for the given connection
       - Extract field names mentioned in glossary term definitions/descriptions
       - Map to canonical field names via FIELD_REGISTRY
       - Return list of available field names for filter extraction hints
       - If DB unavailable or no terms: return empty list (graceful degradation)

    2. `load_value_map(db: AsyncSession) -> dict[str, dict[str, str]]`:
       - Load dictionary entries with value_map JSON from DB at startup
       - Returns nested dict: `{field_name: {user_value: db_value, ...}, ...}`
       - Cache in module-level variable for reuse (no per-query DB hits)
       - If DB unavailable: return empty dict (graceful degradation)

    3. `normalize_value(value: str, field: str, value_map: dict) -> str`:
       - Look up value in value_map[field]
       - If found: return mapped DB value
       - If not found: return value unchanged
       - Case-insensitive matching for user values

    4. `normalize_values_batch(filters: list[FilterClause], value_map: dict) -> list[FilterClause]`:
       - Apply normalize_value to each filter's values
       - Return new list of FilterClauses with normalized values
       - Non-text fields (date, numeric) skip normalization

    5. Create `backend/tests/test_semantic_wiring.py` with tests mocking DB session and dictionary entries.
  </action>
