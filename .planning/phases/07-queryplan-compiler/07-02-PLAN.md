       - Get filters from state (set by extract_filters)
       - Get existing query_plan from state (dict | None)
       - Get base_intent_sql from state params `_prior_sql` or existing plan

    2. Accumulation logic:
       a. If `initial_query_plan` is None (first turn or LLM fallback → domain tool):

            - Date ranges: replace (last-wins)
            - Boolean/scalar: replace (last-wins)
            - Coerce single string to [string] for multi-value fields
       c. If domain/intent differs from existing plan:
          - Create fresh QueryPlan (topic switch)
